# anime-with-her 架构设计 v1（速度优先，角色级分轨强制）

## 1. 文档信息

- 日期：2026-04-09
- 状态：已确认基线
- 目标：在保证角色级分轨与角色音色复刻的前提下，优先缩短出片时间

## 2. 核心决策（已锁定）

1. `角色级分轨` 是基础能力，非可选项。
2. `max_speakers = 4`（单任务默认最多 4 角色）。
3. 运行策略：`速度优先`。
4. LLM 仅介入：`翻译/可说性重写 + 质检(QC)`。
5. 供应商策略：`API 优先`，未来可扩展本地私有化，且接入协议保持一致。
6. 双供应商：`Qwen 主路由`，`豆包容灾回退`。
7. TTS：先走供应商 API。
8. 角色音色目标：复刻原视频角色音色（自动截样入库，人工可校正）。
9. 音色替换后：`整片重跑`。
10. 重跑成功后：默认自动替换为最新成片。
11. 成片保留策略：每个作品仅保留最近 2 个版本。
12. 审计日志：保留 30 天，到期硬删除。
13. 成本控制：单任务预算上限 `¥5`，超限进入人工决策点。
14. 超限等待 10 分钟无响应时，默认执行：`停掉口型同步，仅完成配音成片`。

## 3. 目标与边界

### 3.1 目标

1. 输入日语动漫视频，输出中文配音视频。
2. 保持角色区分和角色音色一致性。
3. 首版结果快速可交付，口型同步可异步补全。
4. 全流程可审计、可回滚、可扩展到本地私有化。

### 3.2 非目标（v1 不做）

1. 无上限角色支持（v1 固定最多 4 角色）。
2. 完整自动角色命名准确率 100% 承诺（允许人工校正）。
3. 自建训练管线（v1 仅做推理与编排）。

## 5. 系统总览

```text
[Web Console]
  ├─ 任务提交 / 进度查看 / 预算决策
  └─ 剧名音色管理（自动入库 + 人工替换 + 一键整片重跑）

[Backend Orchestrator]
  ├─ Job 状态机
  ├─ Provider Gateway（Qwen 主 / 豆包备）
  ├─ 预算与策略引擎（¥5 上限 + 超限决策）
  ├─ 版本管理（仅保留最近2版）
  └─ 审计日志（30天硬删除）

[Worker Pipeline]
  ├─ A. 音频与说话人处理：分离 + diarization + 角色映射
  ├─ B. 文本链路：ASR -> 翻译/重写 -> QC
  ├─ C. 语音链路：角色音色复刻 TTS -> 多轨混音
  ├─ D. 视频链路：快速配音合成（首版）
  └─ E. 口型同步链路（异步、可降级）
```

## 6. Pipeline 设计

### 6.1 主链路（快速交付）

1. 输入标准化（帧率、采样率、编解码）。
2. 人声/背景分离。
3. VAD 切句 + 说话人分离（最多 4 角色）。
4. ASR（含时间轴）。
5. 翻译与可说性重写（LLM）。
6. QC（LLM + 规则）。
7. 角色级 TTS（API 优先，按角色复用音色档）。
8. 多轨混音与峰值保护。
9. 合成 `dub_fast.mp4`（首版可交付）。

### 6.2 异步链路（精修）

1. 口型同步任务入队。
2. 预算允许且策略允许时执行口型同步。
3. 合成 `dub_refined.mp4`。

## 7. 角色级分轨与音色复刻

### 7.1 角色分轨

1. 每句带 `speaker_id/start/end/confidence`。
2. 超过 4 个候选说话人时执行合并策略并打低置信标记。
3. 同一 `speaker_id` 保持同一音色档和参数模板。

### 7.2 剧名音色管理（Series Voice Registry）

1. 自动截样：系统自动从原片抽取高质量角色参考音频并入库。
2. 状态标记：`auto_collected`（自动采集）、`manual_verified`（人工确认）。
3. 人工校正：允许替换参考样本并生成新版本。
4. 替换后流程：触发整片重跑（不是局部重跑）。

### 7.3 音色库数据模型（建议）

- `series`：剧名与基础信息。
- `character_voice_profile`：角色与当前生效音色档。
- `voice_profile_version`：版本快照（样本、质量分、创建者、时间）。

## 8. Provider Gateway（统一协议，可私有化扩展）

### 8.1 设计原则

1. 上层只依赖统一接口，不感知厂商差异。
2. 统一输入输出结构，支持未来本地模型无缝替换。
3. 所有调用都可观测（时延、成本、失败原因、trace_id）。

### 8.2 路由策略

1. 默认：`Qwen -> Doubao`。
2. 触发回退：超时、限流、5xx、结构校验失败。
3. 本地私有化阶段：新增 `Local*Adapter`，不改编排层。

### 8.3 适配器接口（建议）

- `TranslateProvider`
- `TTSProvider`
- `QCProvider`

统一字段建议：
- 输入：`task_id, segment_id, speaker_id, source_text/style_hint/reference_audio`
- 输出：`result, confidence, provider, latency_ms, estimated_cost, trace_id`

## 9. 成本与预算控制

1. 单任务预算上限：`¥5`。
2. `80%` 预算预警，`100%` 触发超限决策点。
3. 决策选项：
1. 停掉口型同步，仅完成配音成片
2. 继续跑完（展示预计追加成本）
4. 等待用户响应 `10 分钟`。
5. 超时默认执行：`停掉口型同步，仅完成配音成片`。

## 10. 版本管理与保留策略

1. 每次整片重跑生成新成片版本。
2. 成功后默认自动替换为当前发布版本。
3. 仅保留最近 2 个成片版本。
4. 超出版本及其中间产物自动清理。

## 11. 审计与日志治理

1. 审计日志保留 30 天。
2. 到期硬删除（无软删除缓冲）。
3. 清理任务每日定时执行。

## 12. 任务状态机（建议）

`created -> running -> partial_done -> done -> failed`

补充状态：
- `awaiting_budget_decision`（预算超限等待用户）
- `rerendering`（整片重跑中）

## 13. 关键质量指标（v1）

1. `speaker_consistency_score`（角色一致性）
2. `clone_similarity_score`（音色复刻相似度）
3. `duration_alignment_error`（句级时长偏差）
4. `audio_peak_violation_rate`（爆音率）
5. `fallback_rate`（供应商回退率）
6. `cost_per_minute`（每分钟处理成本）

## 14. 风险与缓解

1. 风险：多人对话串角色。
缓解：角色置信阈值 + 人工可校正 + 重跑机制。

2. 风险：API 抖动导致任务不稳定。
缓解：双供应商回退 + 幂等重试 + 缓存。

3. 风险：成本失控。
缓解：预算上限、超限决策点、任务级成本账本。
